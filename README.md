Definitions of all the OOPs concepts

Class: A class is a blueprint or template that defines the attributes (data) and methods (functions) that its objects will possess.

Object: An object is a specific instance of a class. It stores actual values for the defined attributes and can utilize the methods provided by the class.

Attributes/Methods: Attributes are variables that store an object's data, while methods are functions that define an object's behavior.

Encapsulation: Encapsulation is the process of wrapping data and methods together in a single class and restricting direct access to some data.

Magic Method: A magic method is a special method with double underscores (__) that defines built-in behavior for objects.

Static Method: A static method is a method that belongs to the class rather than an object and does not access instance or class data.

Inheritance: Inheritance is a mechanism that allows one class to acquire the properties and methods of another class.

Types of Inheritance : 

    * Single Inheritance: A class inherits properties and methods from one parent class.
    * Multiple Inheritance: A class inherits properties and methods from more than one parent class.
    * Multilevel Inheritance: A class inherits from a class that itself inherits from another class.
    * Hierarchical Inheritance: Multiple child classes inherit properties and methods from a single parent class.
    * Hybrid Inheritance: A combination of two or more types of inheritance in a single program.
